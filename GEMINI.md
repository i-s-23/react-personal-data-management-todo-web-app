# プロジェクト概要

これは、ReactとTypeScriptを使用して構築された「Todo」ウェブアプリケーションです。ディレクトリ名から、プロジェクト名は `react-personal-data-management-todo-web-app` と推測されます。

このアプリケーションは、開発サーバーとビルドプロセスにWebpackを使用しています。コードの品質はESLintとPrettierによって維持されています。また、UIコンポーネントを独立して開発・テストするためにStorybookが導入されています。

## ビルドと実行

### 開発サーバー

ローカルの開発サーバーを起動するには：

```bash
yarn start
```

これにより、新しいブラウザタブでアプリケーションが開きます。

### 本番ビルド

本番環境用のビルドを作成するには：

```bash
yarn build
```

成果物は `dist/` ディレクトリに配置されます。

### リンティング

コードに構文エラーがないかチェックするには：

```bash
yarn lint
```

構文の問題を自動的に修正するには：

```bash
yarn lint:fix
```

### Storybook

Storybookコンポーネントエクスプローラーを実行するには：

```bash
yarn storybook
```

これにより、ポート6006でStorybookが起動します。

### テスト

`package.json` には、特定のテストコマンドは設定されていません。

## 開発規約

*   **言語:** 主な言語はTypeScript（`.tsx`）とCSSです。
*   **コンポーネント駆動開発:** Storybookの存在は、コンポーネントを独立して構築し、文書化するという規約を示唆しています。ストーリーファイルは、コンポーネントの隣、または `src/stories` ディレクトリに配置されています。
*   **スタイリング:** 依存関係に `styled-components` が含まれていることから、これがコンポーネントのスタイリングに推奨される方法であることがわかります。
*   **コードスタイル:** ESLintとPrettierが、一貫したコードスタイルを強制するために設定されています。設定は `.eslintrc.json` で確認できます。
*   **パッケージマネージャー:** このプロジェクトでは、パッケージ管理にYarnを使用しています。
